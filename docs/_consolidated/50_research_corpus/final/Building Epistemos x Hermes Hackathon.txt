EPISTEMOS × HERMES: Deep Research Technical Dossier
The architectural integration of Hermes Agent into the Epistemos cognitive substrate represents a fundamental departure from contemporary AI application design. By rejecting the sidecar paradigm and embedding intelligence directly within a strictly typed, native macOS entity graph, the architecture demands rigorous technical execution. The substrate ceases to be mere storage and becomes the absolute runtime of cognition. This dossier operationalizes that vision, dissecting ten critical research domains to architect a sovereign, local-first, Metal-rendered system capable of autonomous, verifiable thought.
D1. The MCP Substrate Surface
The Model Context Protocol (MCP) serves as the exclusive boundary between the Hermes Agent reasoning loop and the Epistemos cognitive substrate. This interface must be executed with absolute minimal friction, exposing a complete linguistic surface without compromising the generational memory safety of the underlying Rust entity graph.
State of the Art
The current state of the art for native Rust MCP servers is defined by the rmcp crate, which provides an official SDK implementation backed by the tokio asynchronous runtime.1 Modern implementations utilize procedural macros (#[tool_router], #[tool_handler]) to automatically generate JSON-RPC 2.0 compliant schemas and dispatch logic.1 For local, single-machine native architectures, standard input/output (stdio) transport is the prevailing standard, drastically reducing the serialization and networking overhead inherent in Server-Sent Events (SSE) or HTTP transports.2 The stdio transport binds the lifecycle of the MCP server directly to the client process, ensuring immediate termination upon client disconnect.2

	Architectural Recommendation
The Epistemos MCP server (epistemos-hermes-mcp) must be compiled as a distinct, standalone Rust binary embedded within the macOS App bundle. Compiling the server dynamically into the Swift host process introduces severe risk; a catastrophic panic within the MCP processing layer would crash the Metal rendering loop and the SwiftUI host. The standalone binary will communicate via stdio transport.
Graph entities in substrate-core are natively managed via a generational arena (slotmap), generating u64 keys.7 Because LLMs and JSON-RPC interfaces mishandle raw 64-bit integers and generational indices, the Rust server must serialize these u64 entity IDs into opaque, base62-encoded string tokens before passing them to Hermes. GraphEvent emissions will be bridged via tokio::sync::broadcast channels, allowing the UI to passively observe MCP-induced state mutations asynchronously.

       Ok(serde_json::to_string(&node).unwrap())
   }
}

D2. Skills as Graph Nodes
Hermes Agent natively curates its procedural memory through a filesystem-based architecture, loading .md skill files from ~/.hermes/skills/.8 To realize the vision of "skills as graph nodes," Hermes must be forcefully decoupled from the local filesystem and wired directly into the Epistemos graph via MCP.
State of the Art
The hermes-agent implementation uses a progressive disclosure pattern for skill loading. The SkillLoader orchestrates this by parsing SKILL.md frontmatter to present a lightweight catalog to the LLM (Level 0), loading the full file content only when explicitly requested via skill_view (Level 1).10 While Hermes features a plugin architecture (OptionalSkillSource) designed for fetching verified skills from GitHub or external registries 9, there is currently no native configuration flag allowing it to query an MCP tool dynamically during the initial system prompt assembly to populate its skill index.

When Hermes autonomously synthesizes a new skill upon task completion, a custom tool override will intercept the filesystem write, redirecting it to graph.create_node(type: "Skill") and forging a learned-in-session edge to the active session subgraph.

import json
import subprocess
from hermes_agent.skills.loader import SkillSource, Skill

class GraphSkillSource(SkillSource):
   def __init__(self):
       self.mcp_binary = "/Applications/Epistemos.app/Contents/MacOS/epistemos-hermes-mcp"

   def get_catalog(self) -> list:
       req = {
           "jsonrpc": "2.0", "id": 1, "method": "tools/call",
           "params": {"name": "query_nodes_by_type", "arguments": {"type": "Skill"}}
       }
       res = self._call_mcp(req)
       return [self._parse_skill(node) for node in res.get("nodes",)]

   def _call_mcp(self, req: dict) -> dict:
       proc = subprocess.Popen(
           [self.mcp_binary], stdin=subprocess.PIPE, stdout=subprocess.PIPE
       )
       stdout, _ = proc.communicate(json.dumps(req).encode('utf-8'))
       return json.loads(stdout.decode('utf-8')).get("result", {})

   def _parse_skill(self, node: dict) -> Skill:
       # Maps graph node payload to Hermes internal Skill representation
       return Skill(
           name=node["name"],
           description=node["description"],
           body=node["body"],
           version=node["version"]
       )

D3. Loop Profiles: The Editable Brain
Loop Profiles elevate Hermes from a stateless reasoning engine into an editable, recurrent cognitive process. Instead of static prompts, the user creates sandboxed, executable reasoning loops stored directly within the substrate.
State of the Art
The native execute_code tool within Hermes Agent utilizes a child process to run Python scripts generated by the LLM. However, recent critical vulnerability disclosures (CVE-7071) reveal that the execute_code Python sandbox is dangerously porous.15 PYTHONPATH injections allow scripts executed within the sandbox to import sensitive internal modules, exposing host environment secrets, API keys, and filesystem configurations.15
Concurrently, the hermes-agent-self-evolution repository demonstrates advanced capability optimization utilizing DSPy and Genetic-Pareto Prompt Evolution (GEPA).16 GEPA reads execution traces, understands failure modalities, and mutates system behaviors. However, this relies entirely on remote API calls and Git-based organisms rather than secure, local runtime mutations.16

	Architectural Recommendation
To guarantee capability honesty and absolute security on a sovereign macOS machine, Epistemos must completely abandon Hermes's vulnerable execute_code Python sandbox. Instead, the native application will utilize Apple's JavaScriptCore (JSC) or a Rust-embedded WASM runtime (wasmtime) as a strictly isolated sandbox for running user-authored Loop Profiles.
A Loop Profile will be stored as a LoopProfile node in substrate-core. When Hermes dictates the execution of a profile (e.g., deepen-thought), it calls an MCP tool: graph.execute_loop(profile_id, target_id). The Rust backend instantiates a sterile JSC or WASM isolate, providing it bounded callback functions that map directly back to the secure MCP six-verb surface.

D4. Schema-Driven UI Synthesis
The interface is summoned, not coded. As Hermes returns structured data via MCP, the Swift presentation layer must dynamically synthesize the interface based on a schema registry, eliminating the bottleneck of manual view implementation.
State of the Art
In SwiftUI, Server-Driven UI (SDUI) is traditionally achieved by parsing JSON payloads and mapping them to predefined view structures.18 However, a critical failure mode in SwiftUI SDUI architectures is the reliance on AnyView for type erasure when rendering heterogeneous component arrays.19 Extensive benchmarking reveals that AnyView destroys SwiftUI's structural identity and view-tree diffing algorithm. This results in the complete recreation of view state, causing severe performance degradation, hitching, and broken animations.20
Modern architectural patterns dictate the use of @ViewBuilder and generic protocols to maintain static typings while allowing dynamic injection, leveraging buildLimitedAvailability concepts to mask type erasure from the compiler.18

Implement a ViewRegistry pattern strictly utilizing @ViewBuilder. Hermes MCP tools will emit JSON payloads conforming to specific schema type identifiers (RawThought, ImplementationPlan, Recall, etc.). Swift will decode these payloads into structs conforming to a universal GraphEntity protocol. The registry uses an exhaustive switch statement inside a macro-generated ViewBuilder to map the decoded type directly to a specific, typed SwiftUI view, completely eliminating AnyView from the render pipeline.

import SwiftUI

// 1. The Protocol Contract
protocol GraphEntity: Codable, Identifiable {
   var id: String { get }
   var schemaType: String { get }
}

// 2. Concrete Decoded Schemas
struct ImplementationPlan: GraphEntity {
   let id: String
   let schemaType = "ImplementationPlan"
   let title: String
   let sections:
}

// 3. The Strict ViewBuilder Registry
@MainActor
struct SchemaRegistryView: View {
   let entity: any GraphEntity
   
   var body: some View {
       buildView()
   }
   
   // Type erasure is resolved at compile time via ViewBuilder
   @ViewBuilder
   private func buildView() -> some View {
       switch entity.schemaType {
       case "RawThought":
           if let thought = entity as? RawThought {
               RawThoughtView(thought: thought)
           }
       case "ImplementationPlan":
           if let plan = entity as? ImplementationPlan {
               ImplementationPlanView(plan: plan)
           }
       case "Recall":
           if let recall = entity as? Recall {
               RecallView(recall: recall)
           }
       default:
           FallbackView(entity: entity)
       }
   }
}

D5. The Embedded Browser
Hermes requires an embedded browser to traverse the web autonomously, ingesting context and bypassing modern bot-detection mechanics to execute complex research tasks.
State of the Art
Hermes Agent natively utilizes the Firecrawl API or an agent-browser backend powered by Playwright and Chromium.23 However, Playwright's default Chromium configuration is actively targeted by Cloudflare, Akamai Bot Manager, and Google OAuth via TLS fingerprinting and CDP leak detection.25 This causes automated research flows to fail silently or become trapped in infinite CAPTCHA loops.25 While tools like Camofox mitigate this via fingerprint spoofing, they introduce heavy, bundled dependencies.26
Natively, macOS provides WKWebView, which leverages the host machine's WebKit engine. Because WKWebView shares the Safari network stack and user state, it inherits seamless pass-through against most anti-bot heuristics, making it invisible to standard WAFs.27

import Foundation

@MainActor
class BrowserActionServer: NSObject, WKNavigationDelegate {
   let webView = WKWebView()
   private var navigationContinuation: CheckedContinuation<Void, Error>?
   
   // Exposed to Hermes via MCP
   func navigate(to urlString: String) async throws -> String {
       guard let url = URL(string: urlString) else { throw URLError(.badURL) }
       
       try await withCheckedThrowingContinuation { continuation in
           self.navigationContinuation = continuation
           webView.load(URLRequest(url: url))
       }

       return dom?? ""
   }
   
   // Delegate callback resolves the async continuation
   func webView(_ webView: WKWebView, didFinish navigation: WKNavigation!) {
       navigationContinuation?.resume()
       navigationContinuation = nil
   }
}

D6. Session Graph as Live Cognition
Epistemos functions as a cognitive substrate where every Hermes action emits a GraphEvent, consumed instantaneously by a high-performance Metal renderer.
State of the Art
Rendering a graph of 10,000+ nodes using standard SwiftUI views is fundamentally impossible at 120fps due to CoreAnimation allocation overhead. The industry standard for performant force-directed graphs in native macOS environments involves offloading the physics simulation entirely to a Metal Compute Shader, and the drawing to Vertex/Fragment shaders.29 The event pipeline must be strictly non-blocking; the CPU orchestrates data transfer, but the GPU handles all spatial recalculations.
Architectural Recommendation
Rust owns the OpLog and the substrate-core. When Hermes triggers an MCP tool (graph.traverse), Rust fires a GraphEvent over a UniFFI callback boundary. Swift receives this in a strictly isolated GraphEventStream Actor, updating a triple-buffered Metal uniform array representing node states (color, scale, emission intensity).
The Metal render pipeline utilizes three specific shader uniforms driven by these events:
1. pulse: Increases a localized scale multiplier and alpha uniform on a specific vertex during get_node.
2. flash_edge: Interpolates an emissive color across the fragment shader of a connecting line during traverse.
3. phase_in: Modifies a specific node's alpha from 0 to 1 over time while triggering a one-time glare pass during create_node.
Code Stub
Metal Vertex and Fragment Shader (GraphNodes.metal) handling event-driven pulses:


Code snippet




#include <metal_stdlib>
using namespace metal;

struct NodeVertex {
   float2 position [[attribute(0)]];
   float  pulse_intensity [[attribute(1)]];
   float  phase_progress [[attribute(2)]];
};

struct RasterizerData {
   float4 position [[position]];
   float pulse;
   float phase;
};

// Vertex shader applies physical scale transformations based on GraphEvents
vertex RasterizerData nodeVertexShader(NodeVertex v [[stage_in]], constant float4x4 &mvp [[buffer(1)]]) {
   RasterizerData out;
   float scale = 1.0 + (v.pulse_intensity * 0.5); // 50% scale increase on pulse
   out.position = mvp * float4(v.position * scale, 0.0, 1.0);
   out.pulse = v.pulse_intensity;
   out.phase = v.phase_progress;
   return out;
}

// Fragment shader mixes colors based on event interpolation
fragment float4 nodeFragmentShader(RasterizerData in [[stage_in]]) {
   float4 baseColor = float4(0.96, 0.95, 0.92, in.phase); // Epistemos Off-white
   float4 pulseColor = float4(0.31, 0.88, 0.82, in.phase); // Accent Cyan
   return mix(baseColor, pulseColor, in.pulse);
}

Risks
1. UniFFI Bottleneck: Bridging thousands of GraphEvents per second across the Rust-Swift FFI boundary will stall the main thread if not batched effectively.
2. Compute Synchronization: The physics compute shader may tear if read/write buffers are not properly synchronized via Metal fences, resulting in flickering nodes.

D7. The Landing Page → App Handoff
The aesthetic transition from the web space into the native application must be immediate, incorporating an ASCII wave, a pixel-art type-on sequence, and a specular glare shader over the Hermes logo.
State of the Art
Transitioning from web to app is achieved via custom URL scheme registration in macOS Info.plist (e.g., epistemos://), handled natively in Swift via the .onOpenURL modifier or Scene delegates.31 For the animation, rendering pure ASCII waves at 60fps within a web browser DOM is highly CPU-intensive and prone to layout thrashing.33 Performing the heavy animation natively within the Metal/SwiftUI environment provides vastly superior hardware acceleration, eliminating web-to-app launch lag and preserving the precise retro-pixel aesthetic.
Architectural Recommendation
The web landing page should remain utterly minimal—capturing the double-click event and immediately triggering the epistemos://launch deep link via JavaScript. The macOS App accepts the warm/cold launch and mounts a transient HermesIntroView overlay.
The ASCII wave will be rendered as a grid of discrete text nodes mapped to a sine function. The Glare animation on the logo is best achieved via a Metal Fragment shader applying a diagonal step function mapped to time (ease-in-out), avoiding heavy video or Lottie dependencies.34
Code Stub
Metal Glare Shader (glare.metal) applying a diagonal specular highlight:


Code snippet




#include <metal_stdlib>
using namespace metal;

fragment float4 glareFragment(float2 uv [[stage_in]], 
                             texture2d<float> texture [[texture(0)]],
                             constant float &time [[buffer(0)]]) {
   constexpr sampler s(filter::linear);
   float4 color = texture.sample(s, uv);
   
   // Diagonal line function: x + y maps the highlight across the sprite
   float diagonal = uv.x + uv.y;
   
   // Time represents progress from 0.0 to 2.0 
   float glareWidth = 0.15;
   float distanceToGlare = abs(diagonal - time);
   
   // Smoothstep for sharp but anti-aliased glare edges
   float glareIntensity = 1.0 - smoothstep(0.0, glareWidth, distanceToGlare);
   
   // Additive blending over original texture
   color.rgb += float3(glareIntensity * 0.8); 
   
   return color;
}

D9. The Aesthetic Stack
The visual register demands retro-pixel-art precision combined with highly disciplined typography and layout constraints—defined as "restraint masquerading as warmth" [User Query].
State of the Art
Rendering bitmap fonts (like Berkeley Mono Pixel or Press Start 2P) in native Apple frameworks via CoreText often falls victim to subpixel anti-aliasing, which destroys the crisp, hard-edged aesthetic of pixel art. For Metal, the industry standard for scalable, perfectly crisp text is Multi-Channel Signed Distance Fields (MSDF).36 However, for pixel fonts specifically, MSDF interpolation creates artificially rounded corners at high scales, ruining the aesthetic.36
Architectural Recommendation
To achieve absolute pixel-perfect text rendering, completely disable MSDF for the pixel-font surfaces. Instead, render text using Point filtering (nearest-neighbor) in the Metal sampler. In SwiftUI, strictly apply .antialiased(false) on Image views, and configure NSFont to disable smoothing at the environment level.
The color palette will be strictly limited to three values, enforcing discipline. These values must be defined in a centralized Tokens.swift file using strict semantic naming.

struct PixelFontModifier: ViewModifier {
   var size: CGFloat
   func body(content: Content) -> some View {
       content
          .font(.custom("BerkeleyMono-Pixel", size: size))
           // Force disable subpixel antialiasing for crisp edges
          .environment(\.fontSmoothing, false) 
   }
}

* Day 10 (May 4): Final deployment and official Nous Discord submission.
Post-Hackathon Trajectory
1. Loop Profiles: Integrate the embedded wasmtime or JavaScriptCore runtime to deprecate the vulnerable Python execute_code sandbox, securing the application for public release.
2. Browser Server: Implement the WKWebView AppKit-bridged MCP server to replace the Cloudflare-blocked Playwright backend, enabling seamless, stealthy web research.
3. Council UI: Implement multi-model dispatching visibility, exposing when Hermes routes tasks to local Qwen vs. cloud Claude.
Cross-Cutting Risks & Reroutes
* Risk: rmcp tokio runtime deadlocks holding the graph RwLock during heavy concurrent MCP requests.
   * Mitigation/Reroute: Decouple the read/write paths using crossbeam channels. If Rust deadlocks persist by May 1, revert to a stateless SQLite polling model for the demo to ensure stability.
* Risk: Metal shaders drop frames during heavy JSON-RPC serialization.
   * Mitigation/Reroute: Move the force-directed physics layout entirely to a Metal Compute Shader. This ensures the CPU only handles coordinate dispatching, completely separating the AI reasoning thread from the 120Hz render loop.
