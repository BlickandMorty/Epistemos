## 1. The recovery sequence (no compromises)

### Stage A — Foundation (must ship before anything else)

**A.1 — Cognitive Kernel Phase 1: Audit fragmentation.**
Produce `docs/fusion/COGNITIVE_KERNEL_AUDIT_2026_05_XX.md` per
`COGNITIVE_KERNEL_DOCTRINE` §Appendix B. This is a grep-powered doc
that names every parallel agent loop, skill registry, procedural
memory store, prompt manager, tool dispatch in the tree, and assigns
a `keep | collapse | reject` verdict. Output is a reading map for
all subsequent work.

**A.2 — Schema-First GenUI Phase G.1: Generalize the schema.**
Promote `Artifact` to `GenUIPayload`, expand `ChatArtifactKind` to
`GenUISchema` with the new variants (`keyValueTable`,
`commandReceipt`, `actionPanel`, `errorReport`, `progressIndicator`,
`capabilityList`, `searchResultSet`, `provenanceTrace`). Spec in
`COGNITIVE_GENUI_DOCTRINE` §4 G.1.

**A.3 — Schema-First GenUI Phase G.2: Build the `GenUIDispatcher`.**
Static registry mapping schemas to renderer types. App bootstrap
registers canonical renderers. Spec in `COGNITIVE_GENUI_DOCTRINE`
§4 G.2.

**A.4 — Schema-First GenUI Phase G.3: Migrate Hermes Expert Mode renderers.**
Deletes every `renderXxxInline` per-command renderer in
`HermesExpertModeRunner` and replaces with `state.append(.payload(GenUIPayload(...)))`.
**This is the single biggest unwind of today's hackathon shortcuts.**

### Stage B — Hermes runtime in Rust (the agent that actually runs)

**B.1 — Cognitive Kernel Phase 2: Hermes-in-Rust.**
Five Rust modules in `agent_core/src/hermes/`:
`prompt_format.rs`, `function_call.rs`, `skills.rs`,
`procedural_memory.rs`, `self_evolution.rs`. Spec in
`COGNITIVE_KERNEL_DOCTRINE` §4. **Result:** the Hermes Expert Mode
UI calls into this canonical runtime instead of fanning out to per-
command stubs. `/ask` routes through the kernel; `/persona switch`
mutates real `agent_core::hermes::persona` state; `/skills list`
enumerates real skills; `/memory recall` returns real procedural
memory rows.

**B.2 — Apple FoundationModels integration.**
The `appleIntelligence` provider variant exists in `InferenceState`;
wire it as a real provider through `agent_core/src/providers/`.
Per the active surface in `MAS_FIRST_FOCUS_DOCTRINE` §1.

**B.3 — Cognitive Kernel Phase 6: Capability lattice consolidation.**
Single capability lattice (`Tier::Core | Pro | Research | Both | All`)
with structural filtering at the registry layer per
`COGNITIVE_KERNEL_DOCTRINE` §8. The Pro-only Hermes commands stay in
the registry but are filtered out of the active dispatcher in MAS
builds (which is what they already do via `parseCore`; this makes
the filter explicit instead of accidental).

### Stage E — Simulation assets (the visual canon)

**2026-05-04 user correction — Tamagotchi means creatures, not icons.**
Stage E is not satisfied by SF Symbols, generic orbs, static cards, or an
abstract "body grammar" checkbox. The intended product surface is small
Tamagotchi-like companion creatures with a simple styleable avatar grammar.
They live on the Landing Farm, wander/idle-walk deterministically inside
bounded paths, and later appear in the graph surface as companion presences.
Every Stage E slice must search for: `tamagotchi`, `companion`, `farm`,
`avatar`, `creature`, `pet`, `body grammar`, `walk`, `roam`, `wander`,
`Landing Farm`, `Graph Live Theater`, and `CompanionView` before coding.

**E.1 — Author `SIMULATION_ASSETS_DOCTRINE_2026_05_XX.md`.**
Spec each body grammar visually:
- **Block:** square outline + slow rotational micro-pulse +
  inner geometric pattern. SwiftUI `Canvas` is sufficient.
- **Sage:** tall robed silhouette with hooded head, vertical
  orientation, gentle float. SwiftUI `Canvas` + a Metal halo shader
  (slot already exists at `Epistemos/Shaders/`).
- **Orb:** sphere with iridescent halo + per-companion accent hue
  rotation. Metal shader for the iridescence.
- **Hermes Snake:** sinuous winged caduceus shape. SVG-equivalent
  Bezier paths drawn in SwiftUI `Canvas` with optional Metal sheen
  pass.

Reference the visual audit chain memory (`reference_visual_audit_chain`)
before authoring — 100+ docs across 12 tiers must be consulted per
the user's existing discipline.

**E.2 — Implement custom-drawn `CompanionView` body renderers.**
Replace SF Symbols with Canvas/Metal renderers per the asset
doctrine. The current `CompanionView.systemImageName` becomes
`bodyKind.renderer(at: timeline, accent: ..., reduceMotion: ...)`.
The first renderer slice must make the creatures recognizable as
Tamagotchi-style companions, not decorative app icons.

**E.3 — Implement deterministic Landing Farm roaming.**
After `CompanionView` renders native bodies, add a roaming/walking layer over
the Landing Farm. Movement is seeded from companion identity/session state,
bounded to the farm, reduce-motion collapses to static pose + state badge, and
the implementation must avoid per-frame allocations and `repeatForever`.

**E.4 — Plan Graph companion presence without touching graph internals.**
Graph companions are a follow-up projection of the same `CompanionRegistry`.
They may orbit/idle near relevant graph nodes, but the first graph slice must
avoid `MetalGraphView.swift`, `HologramController.swift`, and graph
physics/render internals unless a dedicated graph deliberation approves it.
