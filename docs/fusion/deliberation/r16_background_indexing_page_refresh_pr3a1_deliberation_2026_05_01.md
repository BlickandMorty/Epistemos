# R16 Background Indexing Page Refresh PR3A.1 Deliberation - 2026-05-01

## Verdict

Approved for a narrow follow-on build slice after PR3A.

This gate wires saved note pages into the existing Halo Shadow indexer through
the already-emitted `.vaultPageChanged(pageId:)` event. It does not approve a
new FSEvents live watcher, AFM sidecar generation, ETL FFI counters, xattr
marking, security-scoped bookmark changes, or deletion reconciliation.

## Scope

- Extract a shared vault-relative Shadow document-id helper from
  `ShadowVaultBootstrapper` so targeted updates use the same id as the bootstrap
  crawl.
- On `.vaultPageChanged(pageId:)`, enqueue the changed note into the existing
  `ShadowIndexingService` when the Shadow backend is already open.
- Keep the operation non-blocking and record the same Settings diagnostics
  states added in PR3A.

## Acceptance

- Targeted note updates reuse the exact document id generated by the full vault
  crawl.
- Page-save Shadow reindexing is skipped honestly when no Shadow backend is
  open.
- No page body disk read happens in a SwiftUI view body.
- No protected files or Rust engine paths are touched.
