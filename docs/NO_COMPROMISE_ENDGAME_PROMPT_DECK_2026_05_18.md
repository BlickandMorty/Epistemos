# No-Compromise Endgame Prompt Deck - 2026-05-18

## 0. The exciting honest classification

If Epistemos truly works as intended, it is not in the class of "notes app" or even "AI notebook."

It becomes a **native macOS verifiable cognition substrate**: models are engines; Epistemos is the memory, retrieval, permission, provenance, execution, graph, document, and UI substrate that makes those engines accountable.

The stronger product class is still fair: a **sovereign cognitive computer**. But the builder-facing phrase is "verifiable cognition substrate" because it forces discipline: one spine, typed events, governed actions, visible proof.

| Tier | Honest class | What it means |
|---|---|---|
| Core / MAS | Native knowledge operating layer | A Mac-native local memory, note, graph, retrieval, provenance, and agent shell that stays inside public APIs and App Store constraints. |
| Pro | Personal agentic operating environment | Same substrate, plus bounded CLI/MCP/browser/tool execution, richer agent runtimes, Eidos, Agent Runtime v2, and visible governance. |
| Research / Omega | Falsifiable substrate computer | The research instrument for UAS-ACS, five-plane runtime, lattice compression, KV-Direct, EML, SSM/attention interrupt logic, and the 70B local cocktail. |

The punchline: the app is the shell. The substrate is the computer.

Endgame name for the research program: **Active-Support Verified Cognitive Runtime**.

That means a local-first cognitive workspace where notes, models, graph nodes, tool calls, citations, edits, claims, memory pages, and future model components all share a governed address space. The system does not wake the whole vault, whole graph, whole tool universe, or whole model path by default. It selects the smallest active support, checks authority and error budget, executes through the governed substrate, and leaves a visible/replayable witness.

## 2. Can Eidos be built now?

Yes. Eidos V0 is one of the best things to build now because it is product-useful and substrate-aligned without needing the giant speculative gates.

Eidos V0 is:

- deterministic local search fusion;
- closed-citation retrieval;
- vault + `.epdoc` + graph + code + shadow semantic + exact keyword results;
- source IDs that the model is allowed to cite;
- a visible "Retrieved by Eidos" / source trace surface.

Eidos also has a safe current-app form layer:

- canonical object kinds and stable IDs;
- mutation and witness policies;
- visibility classes;
- compatibility mapping to `AnswerPacket`, `ClaimKind`, VRM, `TypedArtifact`, graph nodes, agent events, tools, notes, and memory pages;
- no metaphysical ontology claims.

Eidos V0 is not:

- model training;
- VPD;
- Lean-dependent;
- 70B-dependent;
- web augmentation by default;
- a new philosophy UI.

If delivery must split, build retrieval first because it fixes the user-facing vault recall wound. Build the form layer next, or alongside retrieval when it can share source IDs and object canonicalization without widening scope.

## 5. The no-compromise end goal

The end goal is not "more features." It is:

> A native Mac cognitive computer where every artifact has an address, every claim has evidence, every action has authority, every model has a bounded role, every memory tier is explicit, every research claim has a falsifier, and every user-facing surface can explain why it did what it did.

Expanded theorem cocktail:

1. **Density law**: Morph/EML can approximate compact controller policies where the formal domain permits it.
2. **Address law**: every cognitive object has a stable UAS/UASA address independent of residency.
3. **Active-support law**: only the relevant slice of notes, graph, memory, model, tools, and agent state should wake.
4. **Lattice-error law**: every compressed or approximate representation pays into WBO.
5. **Glue law**: local context must cohere before it becomes global context.
6. **Duplex law**: hard compact and soft page-backed branches are both allowed, but routing error is accounted.
7. **Witness law**: every meaningful action is typed, permissioned, logged, replayable, and visible.

That is the line. No compromise means the product can be beautiful and emotional, but the substrate stays honest.

Shortest builder commandment:

> Preserve wide, build narrow. Current-app value first: product ledger, vault recall, Eidos V0, closed citations, one native governed agent, UAS/UASA metadata, lattice/WBO register, ACS admission, and M2 Pro falsifier handbook. Keep model surgery, active rank-one runtime, PCF runtime acceleration, and 70B execution Vault/Research-only until falsifiers pass.
