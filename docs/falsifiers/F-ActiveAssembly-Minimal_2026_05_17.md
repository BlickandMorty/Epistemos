# F-ActiveAssembly-Minimal

> Gate #6 in the §4.G falsifier ladder. **On a tiny model or synthetic packet graph, active packet selection
> avoids executing irrelevant assemblies while preserving output within bound.** The first runtime proof that
> "the brain does not ping every neuron."

## §1. Why this gate exists

The §4.G hierarchy puts the Active Assembly Runtime (AAR) directly under ACS as the NERVOUS SYSTEM: it decides
which packets / components / model mechanisms fire for the current state. Without a measured proof of selective
firing, "active assembly" is rhetoric. This gate operationalizes the claim that the runtime can skip irrelevant
work while preserving output quality within a known bound.

Driver §4.G prose:

> **F-ActiveAssembly-Minimal** — on a tiny model or synthetic packet graph, active packet selection avoids
> executing irrelevant assemblies while preserving output within bound. This is the first runtime proof that
> "the brain does not ping every neuron."

This gate lands the Phase B.G.B6 target: `agent_core/src/research/active_assembly/` (gap as of 2026-05-17,
audit §C row #25 + §C gap-list).

## §2. The synthetic packet graph

The harness lands a tiny synthetic packet graph in `agent_core/src/research/active_assembly/synthetic.rs`:

- **Nodes (packets)**: `N = 1024` packets, each a `(packet_id, input_pattern, output_pattern, cost_units)`
  tuple. `input_pattern` and `output_pattern` are 64-dim bit vectors. `cost_units` is a uniform random `1..16`.
- **Edges**: each packet declares 1-4 predecessor packets it consumes from. The graph is a DAG (verified at
  construction).
- **Queries**: `Q = 100` ground-truth queries, each a `(query_pattern, expected_output)` pair generated by
  pre-running the full DAG and recording which packets actually contributed mass > τ to the output (τ = 0.05).

The active-pull selector at `agent_core/src/research/active_assembly/selector.rs` (also Phase B.G.B6) decides
which packets fire for each query. The selector strategy under test is **margin-anchored greedy-pull**:

```text
For each query q:
  active = {output_sink}
  loop:
    candidates = ∪{predecessors(p) for p in active}
    rank candidates by (anchor_score(q, candidate) - cost_weight * candidate.cost_units)
    promote top-K from candidates into active
  until: active set stable OR depth_budget exhausted
  emit: run only packets in active
```

`anchor_score(q, c)` uses the ACS `AcsAnchor` of the candidate packet (gate #3 dependency): the score is the
cosine between `q.pattern` and `c.input_pattern`, modulated by `c.tier` (Current App tier gets +0.1 prior,
Verified Floor gets +0.05).

## §3. Pass/fail recipe (the test that decides)

A `#[test]` in `agent_core/tests/active_assembly_minimal.rs` (lands in Phase B.G.B6) runs:

```rust
let graph = SyntheticPacketGraph::seeded(N=1024, max_predecessors=4, seed=0xAA51);
let queries = GroundTruthQuerySet::generate(graph, Q=100, mass_threshold=0.05, seed=0xBB52);

// Baseline: run ALL packets for every query.
let baseline = queries.iter().map(|q| {
    let trace = graph.run_all(q);
    (trace.output, trace.cost_units_consumed, trace.packets_fired.len())
}).collect::<Vec<_>>();

// Active-pull: run only selected packets for every query.
let active = queries.iter().map(|q| {
    let trace = graph.run_with_selector(q, MarginAnchoredGreedyPull::default());
    (trace.output, trace.cost_units_consumed, trace.packets_fired.len())
}).collect::<Vec<_>>();

// Metrics
let output_bound_violation = baseline.iter().zip(&active)
    .filter(|((b_out, _, _), (a_out, _, _))| hamming(b_out, a_out) > 4 /* of 64 bits */)
    .count();

let cost_ratio = total(active.cost) as f64 / total(baseline.cost) as f64;
let firing_ratio = total(active.packets_fired) as f64 / total(baseline.packets_fired) as f64;

// Gates
assert!(output_bound_violation == 0,
    "F-ActiveAssembly-Minimal FAILED: {} queries diverged from baseline > 4 bits", output_bound_violation);
assert!(cost_ratio < 0.40,
    "F-ActiveAssembly-Minimal FAILED: cost ratio {} ≥ 0.40 (selector is not saving work)", cost_ratio);
assert!(firing_ratio < 0.50,
    "F-ActiveAssembly-Minimal FAILED: firing ratio {} ≥ 0.50 (selector is firing > half the graph)", firing_ratio);
```

Gate **fails** if any of the three assertions fails. Gate **passes** when output stays within 4-bit Hamming
of baseline AND cost is < 40% of full-fire AND fewer than half the packets fire.

### §3.1 Within-bound definition

"Within bound" = output Hamming distance from baseline ≤ 4 bits of the 64-bit output vector. This corresponds
to 93.75% per-bit agreement. The bound is loose enough to allow selector-induced rounding but tight enough
that a quarter-of-the-bits-different selector would fail.

For tiny-model variant (an alternative the gate spec also supports), the bound is `KL/token ≤ 0.05` against
the full-fire model run — same shape as F-ShadowFirst-PageEscalation but on the AAR selector instead of the
page escalation policy.

## §4. M2 Pro 16 GB budget

| Metric | Budget |
|---|---|
| **Output-bound violation count** | 0 out of Q = 100 queries |
| **Cost ratio** (active vs full-fire) | < 0.40 (selector must save ≥ 60% of work) |
| **Firing ratio** (active vs full-fire packet count) | < 0.50 (fewer than half the packets fire) |
| **Selector overhead** (selection time / firing time) | < 5% (selector itself must be cheap) |
| **Wall-time per query (p99)** | < 10 ms |
| **Peak RAM** | < 100 MB (the graph is tiny) |

The cost-ratio and firing-ratio gates are the substantive ones: a selector that picks every packet trivially
preserves output but does no work. A selector that picks none trivially saves work but fails the output bound.
The gate's purpose is to prove both can be true at once.

## §5. Measurement methodology

Same `cargo test --release` + `--test-threads=1` discipline as F-UAS-ZeroCopy-Spine §4. Specific to this gate:

- **Deterministic seeding**: graph seed (`0xAA51`) and query seed (`0xBB52`) are logged. Failure includes both
  seeds in the BLOCKER commit.
- **Per-query attribution on failure**: when the output-bound violation fires, the harness emits the offending
  query, the baseline's fired packet set, the active-pull's fired packet set, and the symmetric difference.
- **Selector-strategy substitution**: the test takes a generic `S: PacketSelector` so alternative selectors
  (e.g. random-K subset, full-fire control, top-K-by-cosine without margin) can be run as A/B against the
  primary selector. Mainly a debug feature; the gate uses `MarginAnchoredGreedyPull::default()`.
- **Cost-vs-bound sweep**: when the gate fails, the harness automatically sweeps `cost_weight ∈
  {0.5, 1.0, 2.0, 4.0}` and reports a Pareto frontier (cost_ratio, output_bound_violation_count). Confirms
  whether the failure is fundamental or merely a tuning miss.

## §6. Fallback if the gate fails

Per §4.G "No silent skips":

1. **Identify the failure mode**.
   - **output_bound_violation > 0 AND cost_ratio low**: the selector is too aggressive at pruning. The
     anchor_score is missing relevant packets.
   - **output_bound_violation == 0 AND cost_ratio ≥ 0.40**: the selector is too conservative. It saves the
     bound but doesn't save work.
   - **Both fail**: anchor_score is poorly conditioned; the selector can't tell signal from noise.
2. **Mitigation tier** (least invasive first):
   - **Tier 1 — cost-weight tune**: re-run with adjusted `cost_weight`. Check the Pareto sweep from §5.
   - **Tier 2 — anchor-score refinement**: add a `tier` prior (already in spec; verify it's actually being
     applied) or a `theorem_tag` prior (E1/E2-anchored packets get higher priority).
   - **Tier 3 — depth-budget extension**: increase the iteration `depth_budget` in the selector loop.
     Trade: more selector overhead.
   - **Tier 4 — selector swap**: replace `MarginAnchoredGreedyPull` with a different strategy (e.g.
     `ReverseTopologicalCascade` or `LearnedClassifier`). Out-of-scope for v1 unless threshold/anchor tuning
     can't recover.
   - **Tier 5 — STALLED**: file STALLED row #26 in canonical-doctrine §5 + BLOCKER commit. Do not push.
3. **Document the mitigation**: `// F-ActiveAssembly-Minimal: cost_weight=2.0 + tier prior +0.15 lifts
   selector above threshold; see docs/falsifiers/F-ActiveAssembly-Minimal_2026_05_17.md §6.`
