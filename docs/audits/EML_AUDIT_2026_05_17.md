## ¬ß2. The math identity (paper-pinned)

**EML primitive** (`operator.rs:21-33`):

```text
eml(x, y) = exp(x) ‚àí ln(y)
```

with principal-branch `ln` over the positive reals; `y ‚â§ 0` and non-finite
intermediates are rejected with `EmlError::{NonPositiveLogArg, NonFiniteResult}`.

**Sources** (cited verbatim from `eml/mod.rs:6-10`):

- Odrzywo≈Çek, *"Liouvillian-elementary universality of `eml(x,y) = exp(x) ‚àí ln(y)`"*,
  arXiv:2603.21852 ‚Äî universality proof on the Liouvillian-solvable subdomain.
- Stachowiak, *"Abelian-group + functional-inverse decomposition for EML"*,
  arXiv:2604.23893 ‚Äî structural decomposition.
- Internal anchor: `docs/HELIOS_V6_1_NEW_RESEARCH_INTEGRATION_2026_05_16.md`
  ¬ß"Terminal B" Phase B.0.

**Partials** (`operator.rs:39-57`):

- `‚àÇeml/‚àÇx = exp(x)` ‚Äî `eml_partial_x` (ignores `y` by symmetry of signature).
- `‚àÇeml/‚àÇy = -1/y` ‚Äî `eml_partial_y` (rejects `y ‚â§ 0`).

**Inverse** (`operator.rs:64-81`):

- `eml_inverse_x(z, y) = ln(z + ln(y))` ‚Äî back-solve for the `x` that yields
  a given output. Rejects `z + ln(y) ‚â§ 0`.

## ¬ß6. Hard fences ‚Äî what EML IS and what it is NOT (the ¬ß5.0 reconciliation row)

The ¬ß4.B prompt text uses the phrase *"energy-based modeling as substrate
primitive"* and the candidate sites name *"energy as confidence proxy"* /
*"energy gradient damps"* / *"energy-weighted re-ranking"* / *"energy as
anomaly signal"*. **The code is the authority** (¬ß0 rule 4).

### ‚úÖ EML IS:

- A **Liouvillian-elementary** binary operator `eml(x, y) = exp(x) ‚àí ln(y)`
  with the universality result of Odrzywo≈Çek arXiv:2603.21852 *over its
  domain*.
- A grammar `S ‚Üí 1 | eml(S, S)` whose terms encode every elementary
  function on the Liouvillian-solvable subdomain.
- A **deterministic f64 substrate** with an fp16-ULP acceptance harness
  (‚â§ 2 ULP bar, `ulp_oracle.rs:44`) that gates the AnswerPacket schema
  freeze (`gate.rs:64`).
- A symbolic-algebra primitive suitable as a **potential / scalar field**
  over a state that can be encoded into one or more `(x, y)` arguments.

### ‚ùå EML is NOT:

- A LeCun-style **Energy-Based Model** (EBM) ‚Äî there is no sampler, no
  Langevin dynamics, no Z partition function, no contrastive divergence,
  no training loop. The acronym collision with "energy-based modeling"
  in the ¬ß4.B prompt text is **terminological**, not architectural. Any
  doctrine row that calls EML an "EBM" is drift.
- A **diffusion / score-matching model** ‚Äî there is no noise schedule, no
  reverse process, no score network. `eml_partial_x` / `eml_partial_y`
  are analytic partials of the binary operator, not score functions of
  a learned distribution.
- A **Hopfield / RBM** ‚Äî no recurrent dynamics, no Gibbs sampling.
- A **probabilistic model at all** ‚Äî `eml(x, y)` returns an exact f64,
  not a likelihood. Treating it as a confidence proxy requires a calibrated
  monotone transform (defined per-site, not in `eml/`).

### The "EML potential" convention (T7 lock)

When a T7 doctrine row or integration site uses the word *"energy"* or
*"potential"*, it MUST mean: **the f64 value of an `EmlExpr` evaluated by
[`evaluate`]**, optionally post-composed with a documented monotone
normalization (e.g. a logistic squash for confidence overlays). Every
integration site MUST define its own state-to-`(x, y)` encoding map and
its own normalization explicitly. No floating EML claims.

This row is the ¬ß5.0 reconciliation between the ¬ß4.B prompt's "energy"
vocabulary and the eml-module-on-disk's elementary-universality semantics.
Code wins; doctrine adapts.
